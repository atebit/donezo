- Redesigned board group cards to "floating" style: `12px` horizontal margin on all card rows, `6px` rounded corners on first/last rows, elevated `--color-surface-info` background (lighter than page surface in dark mode), colored left border (`3px solid var(--group-accent)`). Removed group-level bulk-select checkbox from `GroupHeaderRow`. Removed `boxShadow` inset accent stripe from `TaskRow` (replaced by card wrapper's left border). Removed `border-t-[2px]` accent from `GroupColumnHeader` and `GroupFooter`. Added `px-3` to `StickyHeader` to match group card horizontal indent.

## Decided

- **Floating group cards use `overflow: clip` not `overflow: hidden`** — `overflow: hidden` would clip sticky-positioned title cells during horizontal scroll; `overflow: clip` cuts visuals at the border-radius without creating a scroll container, so `position: sticky; left: 0` still resolves to the virtualizer's scroll ancestor. Supported Chrome 90+, Firefox 81+, Safari 16+.
- **CSS custom property override for group card surface** — setting `--color-surface: var(--color-surface-info)` on the card wrapper means every sticky cell that already uses `bg-[color:var(--color-surface)]` automatically inherits the elevated surface, with no per-component changes needed.
- **Single sidebar, no icon rail** — the original two-panel layout (narrow dark rail + wider workspace panel) was redundant once the top bar also had search/bell/avatar. Merged into one panel using `--color-surface-rail` background. The 30px collapsed state hides all controls when collapsed; acceptable for now, can add icon-only collapsed mode later if needed.
- **Topbar kept but stripped** — rather than removing it entirely, kept the topbar as a slim breadcrumb/navigation strip. Removes the UUID-in-breadcrumb bug surface area but does not fix it yet (breadcrumbs still fall back to boardId when `BoardProvider` is not in scope above `Topbar`).
