- **Removed dead code from BoardTable** — deleted the `GroupTriStateCheckbox` function (~55 lines) and its `Checkbox` import from `@base-ui/react/checkbox`, left over after the epic-16 removal of group-level bulk-select. Updated the stale row-order comment in `renderRow`.

- **Fixed disconnected aggregation / footer row** — the `GroupFooter` (task-count + per-column aggregation row) rendered with a dark gap above it and an empty title-column slot. Root causes: (1) `cardRow` wrappers had no explicit height so the virtualizer row's dark background showed through in the slack space; (2) the footer's title-column slot was an empty `aria-hidden` div. Fixes: added `height: "100%"` to all `cardRow` wrapper divs; moved the 14 px inter-group gap from `marginBottom` on the last card row to `paddingTop: 14` on the `group-header` virtualizer row (absolute rows can't overflow their slot, so marginBottom was silently clipped); updated `GroupFooter` title-column slot to show "N tasks / 1 task" in muted text.

- **Fixed horizontal scroll — sticky header alignment** — at medium viewport widths the board table broke: columns weren't horizontally scrollable and the sticky column-header desynchronized from the data rows. Two-part problem and fix:
  1. TanStack Virtual's absolutely-positioned rows don't contribute to `scrollWidth`, so no horizontal scrollbar appeared. Fix: wrapped the spacer div (and the header) in an inner `div` with `style={{ minWidth: "max-content" }}` — the browser computes the intrinsic width from StickyHeader's CSS grid tracks, establishing scroll width.
  2. StickyHeader must live *inside* the scroll container (not as a sibling) so it shares the horizontal scroll surface and stays column-aligned at any `scrollLeft`. Moved `<StickyHeader />` from a sibling of `<TableVirtualizer>` to the new `header` prop on `TableVirtualizer`, which renders it inside the scroll div before the spacer.
  - Decision: board table columns are **never responsive** below mobile (≤ 768 px). The table scrolls horizontally at all non-phone viewport widths; no column stacking or hiding until phone breakpoint.

- **Fixed card left-border column alignment (3 px offset)** — using `borderLeft: "3px solid var(--group-accent)"` with Tailwind's `box-sizing: border-box` reset ate 3 px of the card's content area, shifting all grid columns 3 px right relative to StickyHeader. Replaced with `boxShadow: "inset 3px 0 0 0 var(--group-accent)"` which draws inside the existing box without affecting layout.

Stripped the class from all 23 files via a global perl replace. Also removed the `border border-dashed border-[color:var(--color-border-strong)]` from `EmptyCellTile` (used by status/priority empty states), which wasn't caught by the same pattern.

- **Moved "Add column" to toolbar, removed StickyHeader and Group button** — the global column-header row (`StickyHeader`) was redundant given that every group already shows its own `GroupColumnHeader`. Removed it entirely; `header` prop on `TableVirtualizer` is now unused/absent. Moved the `AddColumnButton` into `ViewToolbar` (after Hide), restyled from a square icon-only h-9 button to a text+icon toolbar button matching h-8 / `toolbarBtnCn` style. - **Fixed horizontal scroll regression after StickyHeader removal** — removing `StickyHeader` from inside `TableVirtualizer`'s scroll container inadvertently broke horizontal scrolling again. Root cause: `min-width: max-content` on the inner container only works when there is an in-flow element (StickyHeader's fixed-px CSS grid) for the browser to measure. Without it, `max-content` resolves to ~viewport width, the absolutely-positioned rows inherit that width via `width: 100%`, and the `minmax()` grid tracks shrink to their minimums. Fix: added a `tableMinWidth?: number` prop to `TableVirtualizer`; `BoardTable` computes it as `CHECKBOX_TRACK_W (32px) + getColumnWidth(titleColumn) + Σ getColumnWidth(otherCols)` and passes it down. The inner container now always has an explicit pixel floor wider than the viewport when the table has multiple columns, restoring horizontal scroll independently of any in-flow header element.

- **Fixed group header and add-task footer scrolling off-screen during horizontal scroll** — `position: sticky; left: 0` does not work inside TanStack Virtual's rows because each row has `transform: translateZ(0)` for GPU compositing. That transform creates a local coordinate context; the sticky threshold fires against the local origin (always 0) rather than the scroll container's `scrollLeft`. Fix: added an `onScroll` handler to the scroll container that writes `scrollLeft` to a `--table-scroll-x` CSS custom property. Group headers apply `translateX(var(--table-scroll-x, 0px))` (merged with dnd-kit's transform). The add-task footer applies `translateX(max(0px, calc(var(--table-scroll-x, 0px) - 12px)))` to account for the 12 px card margin. Both elements now stay anchored at the viewport's left edge while data columns scroll freely.

- **Fixed endless horizontal over-scroll** — the `translateX` on group headers pushed painted content past the inner container's right edge. Browsers factor translated content into `scrollWidth` calculations, so more `scrollLeft` → more translateX → wider scroll area → a feedback loop. Fix: added `overflow: "clip"` to the spacer div in `TableVirtualizer`. `clip` paints content to the spacer's boundary without creating a scroll container, so it doesn't break `sticky top-0` on group header rows.

- **Fixed 1 px column-border misalignment between column headers and task rows** — `GroupColumnHeader` used `border-r` on each cell while `TaskRow` used `border-l`. Under `box-sizing: border-box`, `border-r` draws 1 px inside the right edge of the grid track; `border-l` on the adjacent cell draws 1 px inside the left edge of the next track. Those are 1 px apart. Fix: changed `GroupColumnHeader` to use `border-l` on all data columns (same position as `TaskRow`), and removed `border-r` from the title column (the first data column's `border-l` provides the divider, exactly as it does in `TaskRow`). The checkbox track retains its `border-r` as it is the only divider there in the header row.

Added `border-b border-[color:var(--color-border-strong)]` to match the editing state and the task rows above it.

The toggle button is now a sibling of the aside inside this wrapper, free to hang 10 px past the right edge without being clipped.

## Decided

- **Inter-group gap lives on the group-header row, not the last card row** — padding-top on the header row is the only safe place for the gap because absolute-positioned rows cannot visually overflow their virtualizer slot. The 14 px gap appears between the rounded card bottom and the next group's header text, which is the correct visual rhythm.
- **Explicit `tableMinWidth` is the canonical horizontal scroll pattern, not `max-content`** — `min-width: max-content` is fragile because it depends on the presence of an in-flow element wide enough to drive the measurement. After StickyHeader was removed, `max-content` collapsed to viewport width. The robust pattern: compute total column width in px in `BoardTable` and pass it as `tableMinWidth` to `TableVirtualizer`, which applies it as `style={{ minWidth: tableMinWidth }}` on the inner container. This is explicit, reactive to column resize/hide, and has no dependency on DOM structure inside the scroll container.
- **Cell borders are the table's responsibility, not the cell's** — individual cell components render their content only; border-between-cells comes from the column-separator classes on the row container (`border-l` on non-title column slots in `TaskRow`). This avoids double borders and lets cells be reused outside the table context.
- **Group button removed from toolbar** — grouping is a view-level config, not a per-session toggle. Removing it reduces toolbar noise; if grouping needs to be surfaced again it belongs in view settings, not the day-to-day toolbar.
- **`AddColumnButton` is now toolbar-only** — StickyHeader was its only other host; restyling the component directly (rather than adding a variant prop) is the right call since there's only one usage context.
- **`overflow: clip` on card wrappers** — `overflow: hidden` would create a new scroll container, breaking `position: sticky; left: 0` on title cells during horizontal scroll. `overflow: clip` cuts visuals at the border-radius without creating a scroll ancestor. Supported Chrome 90+, Firefox 81+, Safari 16+.
- **`overflow: clip` on the virtualizer spacer div caps scroll width** — `clip` prevents translated content from extending the browser's computed `scrollWidth`, unlike `overflow: hidden` (which would create a scroll container and break `sticky top-0`). This is the correct choice whenever you want visual clipping without side-effects on sticky/scroll behaviour.
- **`sticky + left: 0` is broken inside transformed ancestors** — `transform: translateZ(0)` on virtualizer row divs creates a local coordinate system. `sticky left` thresholds fire against the local origin, not the scroll container's `scrollLeft`. Use the `--table-scroll-x` CSS custom-property approach (JS `onScroll` → translateX) for any element that needs to stay anchored during horizontal scroll inside a transformed virtualizer row.
- **Column border alignment rule** — use `border-l` consistently on all data cells (header AND task rows) for column separators. `border-r` and `border-l` on adjacent cells are 1 px apart due to box-sizing, causing visual misalignment. Never mix `border-r` on one component with `border-l` on another to draw the same logical divider.
- **Sidebar toggle pill must live outside `overflow: hidden` ancestor** — the pill hangs 10 px past the sidebar's right edge. Placing it inside the `<aside>` (which clips content) always clips it. Pattern: wrapper `<div position:relative, width transition>` → `<aside overflow:hidden>` (content) + `<button position:absolute right:-10>` (toggle). The wrapper has no overflow constraint so the pill is always visible.
