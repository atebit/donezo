## Learned

- **When generated Supabase types are regenerated, grep every synthetic Row construction, not just the ones the next build flags.** Vercel stopped at the first error (`BoardActivityTrigger`) but four other files had the same `last_view_per_board`-missing pattern. They'd have failed the next build. Pattern to grep when adding a column: `<existing required field>: null` or `: \"...\"` blocks inside the affected Row's likely call sites.

- **`EMAIL_FROM` env var is load-bearing, not decorative.** Code falls back to a hardcoded apex (`donezo.app`) that won't be a verified Resend sender on a different domain. Sends silently fail. The runbook needs to call this out as a required-not-optional override per environment.

- **Resend's "Add Domain" verification is independent of the web app's DNS.** Verifying `www.dnzo.app` for the web app doesn't help Resend; Resend needs SPF + DKIM (and ideally DMARC) on whatever sender domain you put in `EMAIL_FROM`. Worth a sentence in the board-invitations runbook.
